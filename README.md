### Git 基本操作指南

对于非程序员的小伙伴，这里提供最简单的 Git 操作步骤：

#### 1️⃣ 首次使用：克隆仓库到本地

```bash
git clone https://github.com/ethanone/cdkrsd-ai-site.git
cd cdkrsd-ai-site
```

#### 2️⃣ 日常更新：拉取最新代码（开始工作前必做）

```bash
git pull
```

这会让您的本地代码与 GitHub 上的最新代码保持同步。

#### 3️⃣ 提交修改：保存您的更改到 GitHub

当您修改了文件后，需要三个步骤：

**步骤 1: 查看修改了什么文件**
```bash
git status
```
这会显示哪些文件被修改了。

**步骤 2: 添加修改的文件**
```bash
git add .
```
`.` 表示添加所有修改的文件。如果想只添加特定文件，可以用：
```bash
git add 文件名
```

**步骤 3: 提交修改（写一个说明）**
```bash
git commit -m "修改说明：例如：更新了联系方式信息"
```
`-m` 后面的内容是对这次修改的简要说明，便于其他成员了解您做了什么。

**步骤 4: 推送到 GitHub**
```bash
git push
```
这样您的修改就会上传到 GitHub，其他成员可以看到。

#### 📝 完整工作流程示例

```bash
# 1. 先拉取最新代码
git pull

# 2. 修改文件（使用编辑器或 IDE）

# 3. 查看修改了哪些文件
git status

# 4. 添加所有修改
git add .

# 5. 提交并写说明
git commit -m "更新了公司简介内容"

# 6. 推送到 GitHub
git push
```

#### ⚠️ 常见问题

**如果提示需要先拉取代码：**
```bash
git pull
# 如果出现冲突，联系其他成员协助解决
git push
```

**如果想查看修改历史：**
```bash
git log
```

**如果想放弃某个文件的修改：**
```bash
git checkout -- 文件名
```
